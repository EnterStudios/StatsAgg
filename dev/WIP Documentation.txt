# opentsdb_http_output_module_n : StatsAgg supports outputting to multiple OpenTSDB endpoints. Each 'OpenTSDB via HTTP' endpoint is defined in StatsAgg via a 'OpenTSDB HTTP Output Module'.
#                                 This output format is supported by OpenTSDB 2.0+
#                                   If you are outputting metrics to an OpenTSDB server this module (HTTP), then you should not output to the same server via telnet.
#                                 The format for a 'OpenTSDB HTTP Output Module' is: opentsdb_http_output_module_n = opentsdb_http_output_enabled,opentsdb_http_host,opentsdb_http_port,opentsdb_http_num_send_retry_attempts,opentsdb_http_max_metrics_per_message
#                                   opentsdb_http_output_enabled : Enables sending metrics to opentsdb_http. Valid values are 'true' or 'false'.
#                                   opentsdb_http_url : The url that you want to send OpenTSDB metrics to. Example: 
#                                   opentsdb_http_num_send_retry_attempts : The number of times to attempt to send metrics to opentsdb_http before giving up. A recommended retry count is 3.
#                                   opentsdb_http_max_metrics_per_message : The maximum number of metrics to put in a single JSON package & submit to OpenTSDB via a HTTP POST.  A recommended value is 100.
#                                 Note: the 'n' at the end of 'opentsdb_http_output_module_n' is a number that is unique to that output module.
#                                 Examples: opentsdb_http_output_module_1 = true,http://My-OpenTsdb-Server.com:4242/api/put,3,100
#                                           opentsdb_http_output_module_2 = true,http://My-Other-OpenTsdb-Server.com:4242/api/put,3,100
opentsdb_http_output_module_1 = false,http://opentsdb.some-domain.com:4242/api/put,3,100
